## Executive Summary

The port is in **good shape**: 14/14 adapters and the core CLI are faithful to the frozen Node oracle at the level that matters (symlink target math, marker splice, install/uninstall semantics). Many divergences are deliberate robustness improvements (broken-symlink self-healing, deterministic sorting). **Verified clean (static):** the universal adapter (method-by-method), gemini symlink-target computation, registry parse robustness (graceful on missing/extra fields), windsurf (mode selection, --force, --dry-run, transformer), the panic axis across adapters (no input-controlled `unwrap`/`expect`/`panic`/`unsafe` outside `#[cfg(test)]`), and the negative/error-path CLI tests (FireNotBurning, ManifestMissing, UnknownItem, BundleUnsupported all assert exit code + stderr).

**Biggest real risks (all `[static — unverified]` for runtime effect):**

1. **`?`-abort instead of per-adapter collect** — one adapter error aborts the whole command and flips the exit code; in `gather` it leaves skill symlinks on disk with **no campfire state** (half-lit fire invisible to `tend`). Node collects failures and continues.
2. **`gather` records the wrong `skill_count`** — counted only on `Created` (deflates on re-gather) *and* incremented inside the per-adapter loop (inflates ×adapters). `tend`/state JSON show a wrong number.
3. **Hermes project-scope install writes to `./.hermes`**, which Hermes-the-tool never reads (it reads `~/.hermes`) — content silently unusable when the default (non-`--global`) scope is used.
4. **Path traversal via unvalidated `id` on `uninstall`** (cross-adapter) — `command_uninstall` skips registry validation; a `..`-bearing id reaches `fs::remove_file`. **Shared with Node** (Node's `uninstallAll` also doesn't filter `unknown:true` ids), delete-only, bounded to files the user can already delete — but cheap to guard centrally.
5. **TUI: typing `q` in search mode quits the app** instead of entering the letter — the global quit guard runs before the search-mode char handler; the Node oracle deliberately orders the search branch first.

Parity status: **faithful** with a documented set of intentional improvements (see Intentional section) and a handful of real parity bugs below. No Critical-severity defects — nothing breaks on normal use without a crafted id or self-referential content.

## Detailed Findings

### High

**1. `?`-propagation aborts the whole command; Node collects per-adapter failures**
`lib.rs` install (200), uninstall (247), audit (265), sync (495/511/527/545), scatter (747/766/785), and **gather's agent (404) and team (417)** loops all call the adapter with `?`. A single adapter error (EACCES/EEXIST) propagates to `main`, exits non-zero, and skips every remaining adapter/item. Node's `installAll`/`uninstallAll`/`auditAll` wrap each call in try/catch, push `{action:'failed'}`, and continue (exit 0). Sharpest case: gather's agent/team `?` sites precede `record_gather` (lib.rs:424), so an error there leaves skill symlinks on disk with **no state entry** — `tend` can't see the fire. Notably, gather's *skill* loop (375–388) already uses the safe collect pattern, proving the fix is local and precedented. Audit has a distinct Node behavior worth matching: `auditAll` synthesizes `errors:['Audit failed: …']` per framework so the health check still completes. **Fix:** collect `Result` into a results vec; keep returning `Ok(())`; run `record_gather` even on partial failure. `[static — unverified: exit codes]`

**2. gather `skill_count` is both deflated and inflated**
`installed_skill_count` (lib.rs:356) increments only on `Action::Created` (lib.rs:377) — so a re-gather of already-installed skills records ~0. It also lives *inside* `for adapter in adapters::all()`, and universal/copilot/aider/claude-code all advertise `ContentType::Skill`, so each skill is counted once per detected skill-adapter (a 10-skill team → 20 with claude-code + universal). Node passes `allSkillItems.length` — the deduped set. **Fix:** record `skill_ids.len()`.

**3. Hermes project-scope install lands in `./.hermes` (unusable)**
`target_path` maps only `Scope::Global` → `~/.hermes`; every other scope → `project_dir/.hermes` (hermes.rs:151-157). The default install path sets `scope_of(false) = Project` (lib.rs:136-142,183) and never forces Global for Hermes. Hermes is detect-gated on `~/.hermes/config.yaml` regardless of cwd, so a user with Hermes who runs `install skill foo` (no `--global`) symlinks into `./.hermes/skills/…`, which Hermes never reads; a later `audit --global` reports "No Hermes content installed." Node hardcodes `homedir()` and ignores scope ("Global scope only"). The adapter's self-justifying comment ("real installs pass `--global`") is **not enforced anywhere** and is **not** in the locked list. **Fix:** ignore scope (always `~/.hermes`) behind a `#[cfg(test)]` affordance, or force Global in the adapter. `[static — unverified: filesystem effect]`

**4. TUI: `q` quits during search instead of typing the letter**
`app.rs:134-148` matches `(KeyCode::Char('q'), _) => should_quit = true` *before* delegating to the spellbook handler, whose search-mode branch (`screens/spellbook.rs:784-786`) would append the char. Every query containing 'q' (quarto, qualify, formulate-quantum-problem…) abruptly exits and discards the in-progress search. The Node oracle (`tui.js:461`) deliberately places the search branch first with the comment "must come before clearing to capture chars including q." **Fix:** gate q-quit on `!(screen==Spellbook && search_mode)`, keep Ctrl-C unconditional.

### Medium-High

**5. Path traversal via unvalidated `id` on uninstall (cross-adapter, shared with Node)**
`command_uninstall` builds `Item { id: id.to_string(), …, domain: None }` from the raw CLI arg with **no `resolve_item`/registry check** (lib.rs:216-221), unlike `command_install` (lib.rs:165) which validates against the registry. That id flows into `base.join(&item.id)` (or `find_installed_skill`'s scan) and then `remove_link` = `fs::remove_file` in **claude_code, hermes, pi, opencode, copilot, cursor, windsurf, openclaw, ai_edge, universal**. `Path::join` does not normalize `..`/absolute components, so `uninstall skill ../../something` resolves outside the managed tree and deletes a reachable file. **Bounded:** delete-only (no overwrite), only succeeds on an existing file/symlink the user can already delete, the "attacker" is the user's own typo/paste on a local CLI. **Trap corrected:** this is **NOT** a Node-divergence — Node's `resolveItems` tags unknown names `unknown:true` and `uninstallAll` does *not* filter them, so Node's pi/opencode/etc. adapters exhibit the same `resolve(base,id)+unlinkSync`. It is a **shared pre-existing gap**, not a port regression. Graded Medium-High because it is trivially fixed once, centrally, and closes a real (if user-self-inflicted) deletion primitive. **Fix:** reject ids containing path separators / `..` / absolute prefixes in `command_uninstall` (a single guard fixes all adapters); optionally validate against the registry as install does. `[static — unverified: actual deletion]`

### Medium

**6. condense_skill drops procedure steps after the first On-failure block** (codex/aider/windsurf/copilot consumers)
After a `**On failure:**` line sets `skip_block = true`, the early `if skip_block { continue; }` (transformer.rs:93) fires *before* the reset at 122-128, making the `### `/`**Expected:**` reset dead code; only the next `## ` heading clears the skip. Since every template step carries `**On failure:**`, condense keeps Step 1 and jumps to Validation. condense_skill caps frontmatter at `<=2` but the skip-reset asymmetry still drops content. **Faithful port** — Node has the byte-identical dead reset (`transformer.js:53` before `81-84`), so it's a shared upstream defect, not a regression. **Fix in both:** move the reset ahead of the early skip; add a 2-step skill fixture. `[static]`
> *Related, narrower:* condense_agent toggles frontmatter on any `---` (no `<=2` cap, faithful to Node). **Corrected blast radius:** a stray body `---` does *not* re-open frontmatter; it leaks **one line** per stray `---` (a horizontal rule), not a whole section. Low.

**7. distill_skill step-title parsing diverges from Node on real corpus headings** (ai_edge)
Rust strips `### Step ` then splits on the first `:`, falling back to the whole remainder. Node's `replace(/^### Step \d+:\s*/, '')` only rewrites `### Step <digits>:` and otherwise keeps the full line. Divergent on `### Step 2a:` (deploy-shiny-app), `### Step 4.5:`/`5.5:` (evolve-*, translate-content), `### Step 1.`–`6.` (use-graphql-api, period not colon) — **8 distinct skills** (finding said 6; create-skill + evolve-skill-from-traces also use `### Step N:`). Same skill installed via the two CLIs produces different `.ai-edge/skills/<id>.md`. ai-edge is the only consumer. **Fix:** match Node exactly. `[static]`

**8. Manifest `complexity` filter silently ignored; SkillSummary drops `complexity`/`language`** (manifest, two lenses + copilot seed)
`Entry::Domain { domain, .. }` (manifest.rs:141) discards `complexity` and expands to every skill in the domain; Node passes it to `filterSkills` (`s.complexity === filters.complexity`). So `{ domain: r-packages, complexity: basic }` installs only basic skills in Node but **all** in Rust. Root cause: `SkillSummary` (registry.rs:82-89) extracts only id/description/path/domain, so the data to filter on isn't even present, and any future `--complexity`/`--language` parity is impossible. **Fix:** add `complexity`/`language` to SkillSummary via `str_field`; honor `complexity` in `resolve()`.

**9. CampfireState lacks `#[serde(default)]` → partial/old state.json discards all fires**
`CampfireState.version` and `Fire.{gathered,last_warmed,agents,skill_count}` are required (state.rs:43-52,23-38); `load()` does `from_str(..).unwrap_or_default()`. A valid-JSON-but-version-less state (or a fire missing a required field) fails to deserialize and is replaced wholesale by `default()` with empty fires — **silent data loss**. Node's `loadState` catches only `JSON.parse` syntax errors, so partial objects survive. Inconsistent with the sibling XDG `PersistentState.schema_version`, which *does* have `#[serde(default)]`. The `deserialises_node_shape` test always supplies version, so this path is untested. **Fix:** add serde defaults. `[static — serde required-field semantics]`

**10. opencode audit under-counts installed items** (the one anti-improvement)
`audit` derives "N items installed" from `valid` (links whose target resolves), not `installed.len()` (opencode.rs:243-247). With all links broken it emits no installed line; with some broken it shrinks the count to the healthy subset. Node reports `installed.length` and broken separately. Inconsistent with **hermes**, which correctly uses `installed.len()`. **Distinct from** the gemini/pi/copilot broken-link-*error* improvement and from cursor's *absence* of broken-link detection (three directions — do not merge). **Fix:** report the total; keep broken as a separate error line.

**11. `scatter` (and `init`) hard-require `--root` despite reading only the embeddable registry**
`command_scatter` opens with `root.ok_or(Error::RootNotFound)?` (lib.rs:628) but never reads on-disk content — it builds removal Items with `source_dir: PathBuf::new()` (lib.rs:737). `command_uninstall` proves the removal path needs no root. Registries are embeddable via `include_str!`; `search`/`list`/`detect` pass `root` straight through (embedded when `None`). Requiring `--root` for a pure-removal command is an inconsistency. *(Distinct from the locked "autodetect dropped" decision — this is about falling back to the embedded registry, not on-disk autodetect.)* **Fix:** accept `Option<root>`, embedded fallback.

**12. Markdown table layout uses char count, not display width** (TUI / i18n)
`render_table` computes widths/padding via `chars().count()` (markdown.rs:285,308); ratatui lays out by terminal display width. CJK/East-Asian wide glyphs (2 columns, 1 char) under-measure, so `│` separators and the `─┼─` rule misalign. Reachable: zh-CN/ja skill bodies ship pipe tables, loadable via `--root`. Visual only, no panic. *(Secondary cite `pages.rs:64` is wrong — that file doesn't exist; explicitly hedged in the finding.)* **Fix:** add unicode-width.

**13. `wrapped_height` under-estimates → trailing rows of long bodies unscrollable** (TUI)
`wrapped_height` returns `ceil(chars/width)` per line — hard-wrap rows, a *lower* bound. `Paragraph` uses `Wrap{trim:false}` (word wrap, `>=` that). The estimate is used as the scroll *ceiling* (`max_scroll = rendered - inner.height`), so a lower bound where an upper bound is needed makes `max_scroll` too small and the last word-wrapped rows of long SKILL.md/guide bodies unreachable. `markdown::render` emits one unwrapped Line per paragraph, so over-width input lines genuinely exist. **Fix:** over-estimate (or pre-wrap and count exactly). `[static — magnitude unmeasured]`

**14. `--force` silently deletes a real (non-symlink) user file** (cross-adapter)
`remove_link` is `fs::remove_file` with no symlink-type guard (symlink.rs:47-53); install sites guard with `if is_symlink(&target) || target.exists()`, so a real regular file at the target (`is_symlink` false, `exists` true) is deleted. Matches Node's `existsSync ? unlinkSync` so it's parity-correct, but `--force` overwriting user content deserves a confirmed/documented decision. windsurf dir-mode `fs::write` (windsurf.rs:48-71) overwrites a hand-edited `rules/<id>.md` similarly. A real *directory* is protected by `remove_file` erroring. **Fix (optional):** only remove when `is_symlink`; warn before overwrite.

**15. Guide content type cluster** (correctness + CLI surface, 5 lenses)
Three coupled facts: (a) `resolve_item`'s Guide arm returns `almanac_root.join("teams")` — a copy-paste from Team (lib.rs:126); correct is `guides/` (ai_edge maps Guide→"guides"). (b) The Guide arm performs **no registry validation**, unlike Skill/Agent/Team. (c) `cli.rs` Install help says "guides are not installed" and Uninstall says "skills or agents," yet `Kind` (cli.rs:125-131) accepts `Guides` for both. Harmless today only because no adapter lists Guide in `content_types()` (every adapter Skips it), so install no-ops at exit 0. **Fix:** drop `Guides` from the installable `Kind`, or fix the dir + validate + align help text.

### Low

- **`sourceDir || almanacRoot` fallback dropped** (claude_code.rs:57, copilot.rs:99): Rust passes `item.source_dir` straight to `link_target` (which `canonicalize()?`s it); Node falls back to `almanacRoot/skills/<id>`. Dead today — `resolve_item` always populates `source_dir` on the install path. Latent: a future empty-source install errors instead of falling back.
- **remove_marked_section robustness** (transformer.rs:167-185): finds first start and first end independently, no `end>start` guard; on reversed/duplicate markers the slices cross and garble content (no panic — `find()` offsets are valid UTF-8 boundaries). `has_marked_section` checks only the start marker, so an orphaned end marker creates a split-brain block on the next install. **Faithful to Node** (identical `indexOf` logic). Realistic non-adversarial trigger: a skill whose body documents the marker syntax (e.g. create-skill, or the module's own doc comment). **Fix:** scope the end search to `[start_idx..]`, require both markers; mirror into Node. *(security-class graded the "injection" variant High; demoted to Low here — the trigger is self-referential content on a user-owned file, substring-anchored markers can't inject new markers, only fail to match.)*
- **Ordered lists render as bullets** (markdown.rs:176-184): `Tag::List(_)` discards `Some(start)`; `Tag::Item` always emits `• `. SKILL.md Procedure sections are ordered lists. TUI fidelity, no Node oracle.
- **Hermes skill-uninstall domain scan** (hermes.rs:172-180): `domain:None` triggers a `read_dir` scan of all domain dirs vs Node's fixed `domain||'general'`; non-deterministic on the (unrealistic) duplicate-id case. Robustness improvement otherwise.
- **Hermes agent install hardcodes `agents/<id>.md`** (hermes.rs:74-79) vs Node's registry `path` field. Equivalent today (all agent paths follow the convention); latent if a path is ever nested.

Found by 8 lenses → high confidence this is a deliberate improvement over the oracle. **Keep; document in the divergence ledger.** `[static — Node EEXIST is a runtime hypothesis]`
- **pi absolute symlinks** (pi.rs): predates the relative/global convention — intentional, confirmed. Same for the broader **relative-for-project / absolute-for-global** convention (`symlink.rs:98-107`).
- **cursor `.mdc` legacy install deferred** (cursor.rs): not ported on purpose; uninstall/listing still scan `.mdc` for cleanup. The stale `wrap_as_mdc` doc-comment in `transformer.rs:8-9` and `cursor.rs:11` references a symbol that intentionally doesn't exist — **keep the doc-fix as a one-line nit** (Appendix), frame the behavior as locked.
- **Dry-run `Created`/`Removed` labels** and **`--force` reports `Created` on replace** (codex, opencode, etc.): mirror Node — intentional, confirmed.
- **`detected.is_empty()` dead guard** (lib.rs:172 et al.): unreachable because `Universal::detect` always returns `true`, matching Node's `||true` — intentional, confirmed.
- **uninstall/audit loop all adapters** where install detect-gates: known harmless follow-up #255, not a new bug.
- **`list_installed` sorts by id** (claude_code, windsurf, vibe, openclaw, opencode): deterministic vs Node's `readdir` order — accepted improvement, confirmed.
- **`list_installed` symlink-only filter** (claude_code, gemini, opencode, copilot, cursor, universal, vibe, openclaw, pi — 9 lenses): a real, consistent divergence from Node (which lists every `readdir` entry). It only hides hand-placed *non-symlink* entries the CLI never creates, and propagates into audit counts. **Confirm-and-document as the intended port-wide convention**, or align to Node — not silently. *(Borderline; listed here because the consistency across 9 adapters and "only a symlink is a real install" reasoning point to intent, but it deserves an explicit locked note since it changes audit output.)*

## Test & CI Gaps

- **No assertion of symlink target form** (relative project vs absolute global): every integration test asserts only `is_symlink + resolves`; none calls `read_link` to check relative-vs-absolute. A regression emitting absolute targets for project scope (breaking repo portability) would pass. Only `vibe.rs:344` inline unit test checks `is_relative()`. **Add `read_link(..).is_relative()` after a project install in each adapter test.** (Pins a locked invariant — high value.)
- **No global-scope integration test**: every `ctx()` hardcodes `Scope::Project`; the `Scope::Global` branches of 5 adapters are integration-untested (only openclaw/vibe inline unit tests cover them via HOME redirection). **Add one `#[serial]` global test asserting install lands under `<home>/.<fw>/` with an absolute target.**
- **No cross-fire scatter-sharing test**: `command_scatter`'s `kept_skills`/`kept_agents` branch (lib.rs:672-701) that preserves skills other burning fires need has **zero coverage** — the only scatter test gathers one team alone (so `kept_skills` is always empty) and never asserts symlinks left disk. **Graded High among test gaps:** a regression deleting a shared skill (incl. inherited meditate/heal) wouldn't be caught.
- **No codex/copilot multi-section splice / user-content-preservation test**: codex uninstall test uses an empty AGENTS.md and one section; copilot checks one preamble line. A splice bug eating adjacent content or a sibling block would pass. **Add a 2-block + user-prose round-trip test.**
- **No positive `tend` last_warmed test**: only the negative (dry-run) case exists; Node backdates `lastWarmed` and asserts it advances. A "tend warms nothing" bug would pass.
- **`tend` dry-run test uses a 20ms sleep** rather than backdating — weaker/flakier than Node's 2020-backdate (mitigated in practice by sub-second `Rfc3339`, but still inferior). **Replace sleep with backdate.**
- **No `init` overwrite/idempotency test**: second `init` over a user-edited `agent-almanac.yml` is unpinned (data-loss-adjacent). Shared gap with Node.
