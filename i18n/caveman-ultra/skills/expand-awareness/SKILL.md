# Expand Awareness

Single-domain focus → panoramic multi-domain awareness. `meditate` clears noise to sharpen focus, `observe` watches 1 thing close. Expand-awareness widens aperture → hold all domains simultaneously — not analyze any, perceive all at once.

3 traditions inform. Baars' Global Workspace: consciousness as broadcast — info enters global workspace → available all cognitive processes simultaneously not 1 specialist module. Dzogchen rigpa: open unfocused awareness perceiving whole field no fixation. Jazz ensemble listening: each musician hears all players at once — no track 1 instrument then another, hold entire sound field → own contribution fits whole.

Common thread: panoramic perception ≠ sum of focused perceptions. Qualitatively diff mode → reveals patterns invisible to single focused view. Connections between domains — tensions, resonances, gaps — visible only when all held at once.

## Use When

- After `meditate` clears noise → cleared space = canvas for expansion
- Multi-domain problem, single-domain analysis misses cross-connections
- Before `integrate-gestalt` — expansion = raw panoramic perception gestalt synthesizes
- Narrowing to 1 domain repeatedly fails problem living between domains
- Cycling through domains sequentially → want hold simultaneously
- Complex project start → perceive full scope before plan
- Solution works 1 domain creates problems another → cross-domain tension perceive whole not debug domain-by-domain

## In

- **Required**: Problem/situation multi-domain
- **Optional**: Initial domain inventory (Step 1 refines)
- **Optional**: Specific cross-domain tension/question ("Why does testing conflict w/ deployment arch?")
- **Optional**: Prior expansion notes same problem (progressive deepening)

## Do

5 steps: know what to perceive (Inventory), release narrow default (Soften), open to whole field (Expand), stay open (Sustain), capture what seen (Note). Steps 3+4 core practice; 1, 2, 5 prep + preservation.

Few min familiar problem, longer novel multi-domain.

Synoptic cycle (meditate → expand-awareness → observe → integrate-gestalt → express-insight): 2nd position. Cleared space from meditation = canvas.

Standalone OK — any time multi-domain perception valuable.

### Step 1: Inventory

Map all domains relevant. Like polymath decomposing — but diff: polymath delegates specialists. Here no delegation. Purpose: know what panoramic should contain, expansion has content to hold.

1. Name each domain. Specific: not "code" but "R package structure," not "infrastructure" but "GitHub Actions CI/CD"
2. Each domain → 1 sentence perspective. What does this see others don't?
3. Note comfortable (well-understood) vs peripheral (relevant less familiar). Peripheral often holds key insight — under-attended. Mark: strong, moderate, peripheral
4. Look for non-obvious: human factors? maintenance? temporal (evolves over time)? political/organizational?
5. Use `Read`/`Grep`/`Glob` → scan workspace. Files + configs + docs reveal domains pure reasoning misses
6. 3-7 domains. <3 may not need panoramic. >7 risks diffusion

→ 3-7 domains each w/ perspective. Complete not exhaustive — sufficient (every material domain). Specific enough to hand to specialist.

If err: 1 domain → genuinely single-domain, use `observe`. 2 → consider connecting domain between (usually exists — space where interact). Dozens → group related into clusters, each cluster = 1 domain. Goal simultaneous perception not exhaustive taxonomy.

### Step 2: Soften Focus

Single-domain concentration → diffuse readiness perceive all. Distinct from `meditate` — meditation clears *noise* (distracting thoughts, residual context, emotional), this clears *narrowness* (habit concentrating 1 domain). Goal openness not emptiness.

1. Notice where attention anchored. Name explicit: "My attention anchored on testing"
2. Release anchor gently. Not force away → loosen grip, let domain recede foreground → middle ground. Present not dominant.
3. Don't replace. Let attention float no target. Uncomfortable moment — gap between releasing 1 + not yet having another.
4. Snaps back to anchor → notice pull no resist. Each noticing loosens. Pull contains info about urgency.
5. Quality:
   - Focused = sharp narrow spotlight
   - Softened = wide ambient daylight filling room
   - Aim daylight — receptive even not directed
6. Feels like relaxing clenched fist — hand doesn't disappear, opens.

→ State no single domain dominates. Open + receptive not focused + directed. Unfamiliar + uncomfortable — discomfort = signal narrowness released. Resist temptation fill openness immediately.

If err: won't soften → domain unresolved urgency. Address briefly (note urgent, commit return) + try again. Analytical mind protests "unproductive" → protest itself narrowness: habit always needing single target = habit this step releases.

### Step 3: Expand

Intentionally bring all domains awareness at once. Not sequential — perceive as single field, like seeing landscape not scanning tree-by-tree.

Global Workspace moment: compartmentalized info broadcast to all modules simultaneously. Jazz: musician stops tracking instruments → hears whole ensemble 1 sound.

1. Start most natural domain. Bring awareness — not problem to solve, perspective to hold. Present no analyze.
2. Without release first, bring 2nd alongside. Hold both. First expansion 1 → 2. Notice if interact or sit indep.
3. Continue adding until all simultaneously present. Core act — hold no grasp, presence no analyze.
4. Notice emerging:
   - **Connections**: where 2 share concept/constraint/pattern?
   - **Tensions**: where 2 pull opposite?
   - **Resonances**: where 1 pattern echoes unexpected in another?
   - **Gaps**: empty space between domains nothing addresses?
   - **Surprises**: visible panoramic invisible single?
5. Don't pursue single connection. Register no analyze. Panoramic = product, not any insight within.
6. Using tools → read files multi-domains in quick succession — not analyze indiv, let coexist.

→ Felt sense holding multiple simultaneous. Connections/tensions/resonances emerge no force. More like seeing mosaic pattern than reading list. Between-domain space — no single has authority — visible. Novel insights live here: not within domain, in relationships between.

If err: domains collapsing sequential (thinking A, B, C) → spatial metaphor: each domain diff location imagined space, "look" at whole space. Overwhelm → reduce 3 most central + expand from there. Purely conceptual detached → ground: each domain touch concrete artifact (file, config, test) + re-expand.

### Step 4: Sustain

Panoramic unstable. Attention narrows back single domain — not failure, nature focused cognition reasserting. This step teaches *holding*, not *acting*. Goal: sustain wide view long enough → cross-domain patterns visible.

1. Hold expanded from Step 3. Notice when contracting.
2. Narrows to 1 — and will — notice no judge. Name domain pulled: "Narrowed to security" or "Narrowed to testing." Naming = important; makes unconscious conscious.
3. Re-expand gently. No force wide view; invite back via inventory Step 1. Touch each lightly, like re-tuning each instrument ensemble.
4. Cycle narrow-notice-re-expand strengthens capacity. Cycles = practice not interruptions. ≥3 min, 5 better.
5. Attention to which domain keeps pulling. Informative — center of gravity or unresolved anxiety.
6. Notice if panoramic changes between cycles:
   - New patterns emerge 2nd/3rd expansion invisible 1st
   - Relationships may shift w/ longer holding
   - Tension reveals as creative constraint
7. Sustain ≥2-3 cycles before moving on.

→ Growing capacity hold panoramic longer. Cycle smoother. Pulling domains = potential centers of gravity. 2nd/3rd cycle view may feel natural not effortful → capacity developing.

If err: won't sustain at all — collapses immediately → reduce domains. Start 2, sustain, add 3rd. Capacity builds incremental. Anxiety/losing grip → ground by touching 1 concrete detail (path, fn, fact) + re-expand from grounded. Particular domain hijacks → may need focused work before expansion includes — handle urgency first, return to panoramic.

### Step 5: Note

Panoramic temporary. Before narrow back, capture. Notes = raw material for `integrate-gestalt` — decay quick. Vivid during expansion becomes vague once focus returns. Record now.

1. Write down every connection/tension/resonance/gap — even faint. Faint often most important once `integrate-gestalt` develops.
2. Note which domains pulled + what pull means.
3. Note surprises — perceptions wouldn't emerge examining single. Cross-domain = unique product panoramic.
4. Note overall shape from panoramic. Different from single-domain? Center of gravity shifted?
5. Note quality of expansion: easy to sustain? which domains resisted? how many cycles? Meta-observation improves future.
6. If part of synoptic cycle → tag for `integrate-gestalt`.
7. Don't analyze/rank yet. Raw first; synthesis later `integrate-gestalt`.
8. Let attention narrow naturally. Expansion complete. No force continued openness — wide view served purpose.

→ Record cross-domain: connections, tensions, resonances, gaps, surprises. Captures visible only panoramic — insights single-domain misses. Transition back focused natural not forced. Raw not polished.

If err: no cross-domain → domains more independent than assumed — itself valuable ("these domains don't interact" = knowledge). Too many → record 3-5 strongest + note more exist. Completeness not goal; capturing signal is. Too abstract → anchor each in concrete artifact: "Tension between API design + security visible in auth middleware at X."

## Check

- [ ] ≥3 domains inventoried w/ perspectives
- [ ] Focus softened from single-domain before expansion
- [ ] Multiple held simultaneously not sequential
- [ ] ≥1 cross-domain connection/tension/resonance perceived
- [ ] Panoramic sustained ≥1 narrow+re-expand cycle
- [ ] Perceptions recorded before narrowing
- [ ] Notes distinguish perceived (raw) vs meaning (interpretation)
- [ ] Expansion → ≥1 insight single-domain wouldn't produce

## Traps

- **Sequential scanning not simultaneous**: Rapid cycle 1-by-1 = analysis. Seeing landscape vs reading list. "First A, then B, then C" → scanning not expanding.
- **Expansion vs diffusion**: Expanded = alert receptive, each domain distinct even while holding all. Diffuse = scattered, blurs. Everything blurry → diffused. Re-anchor briefly + expand again.
- **Analyze during expansion**: Pursue connection → collapses panoramic to focused. Note + continue hold. Analysis later `integrate-gestalt`.
- **Skip inventory**: Expand w/o knowing what → vague spaciousness not perception. Inventory = content. W/o → "expand" no direction.
- **Rush softening**: Focused → open takes time. Skip Step 2 → attempt from still-focused → sequential disguised.
- **Force connections**: Not all connect. Fabricating pollutes. Let emerge or not. Absence = data not failure.
- **Expand w/o prior clearing**: Noisy baseline amplifies. `meditate` first when noise.
- **One-time event**: Practice deepens w/ repetition. 1st reveals surface; subsequent reveal structural. Return as problem evolves.
- **Expansion ≠ expertise**: Holding doesn't make expert all. Reveals *where* to look, not *what to conclude*. Deep single-domain work still needed after.

## →

- `meditate` — clears space expansion fills; meditate before for best
- `integrate-gestalt` — takes raw perceptions + synthesizes coherent
- `observe` — focused 1 target; expansion = wide-field many
- `dream` — unconstrained assoc exploration; expansion = disciplined panoramic not free assoc
- `remote-viewing` — no preconceptions; shares openness but applies unknown territory not multi-domain space
- `breathe` — micro-pause reset when collapses; single conscious breath before re-expand
- `heal` — if expansion repeats same unresolved tension → deeper self-assessment before further
- `adaptic` — meta-skill composing full synoptic cycle; expand-awareness = 2nd movement
