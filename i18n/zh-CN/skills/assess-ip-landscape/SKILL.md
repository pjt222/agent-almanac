# 评估 IP 图谱

绘制技术领域的知识产权图谱——识别专利集群、空白空间、关键参与者和自由实施风险。产出战略评估以指导研发方向、许可决策和 IP 申请策略。

## 适用场景

- 在新技术领域开始研发之前（哪些已被主张？）
- 评估竞争者拥有强大专利组合时的市场进入
- 准备投资尽职调查（IP 资产评估）
- 制定专利申请策略（在哪里申请、主张什么）
- 评估新产品或功能的自由实施风险
- 监控竞争对手 IP 活动以进行战略定位

## 输入

- **必需**：要评估的技术领域或产品区域
- **必需**：地理范围（美国、欧盟、全球）
- **可选**：要关注的特定竞争对手
- **可选**：自有专利组合（用于差距分析和 FTO）
- **可选**：时间范围（最近 5 年、最近 10 年、全部时间）
- **可选**：分类号（IPC、CPC）如已知

## 步骤

### 第 1 步：定义搜索范围

建立图谱分析的边界。

1. 精确定义技术领域：核心技术区域、要包含的相邻区域、要明确排除的区域
2. 识别相关分类号（IPC、CPC）
3. 定义地理范围（USPTO、EPO、WIPO 等）
4. 设定时间窗口
5. 将范围记录为**图谱章程**

**预期结果：** 清晰、有边界的范围，足够具体以产出可操作的结果，又足够广泛以涵盖相关竞争格局。分类号已识别用于系统搜索。

**失败处理：** 如果技术领域太广（数千个结果），通过增加技术细节或聚焦特定应用领域来缩小。如果太窄（很少结果），扩展到相邻技术。正确的范围通常产出 100-1000 个专利族。

### 第 2 步：收集专利数据

在定义的范围内收集专利数据。

1. 使用图谱章程查询专利数据库：
   - **免费数据库**：Google Patents、USPTO PatFT/AppFT、Espacenet、WIPO Patentscope
   - **商业数据库**：Orbit、PatSnap、Derwent、Lens.org（免费增值）
2. 系统构建搜索查询：核心关键词 + 技术术语 + 分类号 + 日期范围
3. 下载结构化格式的结果（CSV、JSON）
4. 按专利族去重
5. 记录专利族总数和来源数据库

**预期结果：** 范围内的专利族结构化数据集，已去重并有时间戳。

**失败处理：** 如果数据库访问有限，Google Patents + Lens.org（免费）提供良好覆盖。

### 第 3 步：分析图谱

绘制专利集群、关键参与者和趋势。

1. **集群分析**：按子技术分组专利
2. **关键参与者分析**：按总量、近期申请率、平均引用数等识别前 10 专利权人
3. **趋势分析**：绘制时间窗口内的申请趋势
4. **引用网络**：识别被最多引用的专利（基础 IP）
5. 产出**图谱地图**

**预期结果：** 清晰了解谁拥有什么、活动集中在哪里、图谱如何演进。关键阻断专利已识别。空白空间可见。

**失败处理：** 如果数据集太小无法有意义地聚类，合并为更大的组。

### 第 4 步：识别空白空间和风险

从图谱中提取战略洞察。

1. **空白空间分析**（机会）：少有或无专利申请的区域、已过期的专利族、仅一方申请的活跃区域
2. **FTO 风险筛选**（威胁）：按关键/高/中/低分类
3. **竞争定位**：自有组合相对于竞争者的位置
4. 产出**战略评估**

**预期结果：** 可操作的战略建议：在哪里申请、避免什么、关注谁、哪些风险需要详细 FTO 分析。

**失败处理：** 如果识别到 FTO 风险，此筛选是初步的——不替代专利律师的正式 FTO 意见。将关键风险标记为需法律审查。

### 第 5 步：记录和建议

将图谱评估打包供决策者使用。

1. 编写**图谱报告**：执行摘要、范围和方法论、图谱概览、空白空间分析、风险评估、建议
2. 包含支持数据：专利族列表、集群地图、申请趋势图、关键专利摘要
3. 设置持续监控：定义关键区域新申请的提醒查询

**预期结果：** 完整的图谱报告，支持战略性 IP 决策。基于证据、范围清晰、可操作。

**失败处理：** 如果报告太大，先产出执行摘要并按需提供详细章节。

## 验证清单

- [ ] 图谱章程定义了范围、分类、地理和时间窗口
- [ ] 专利数据集从多个数据库收集并去重
- [ ] 集群已识别并有申请数量和趋势方向
- [ ] 前 10 关键参与者已分析并有组合指标
- [ ] 空白空间已识别并按战略价值排序
- [ ] FTO 风险已筛选并按严重程度分类
- [ ] 关键阻断专利已通过引用分析识别
- [ ] 建议具体且可操作
- [ ] 局限性已承认（筛选 vs. 正式 FTO 意见）
- [ ] 监控提醒已定义用于持续图谱追踪

## 常见问题

- **范围太广**："AI 专利"不是图谱——那是一片汪洋。要具体到技术和应用
- **依赖单一数据库**：没有单一专利数据库有完整覆盖。至少使用两个来源
- **忽略专利族**：计算单个申请而非族会夸大数量。同一发明在 10 个国家申请是一个专利族，不是十个
- **混淆申请与授权**：未决申请不是可执行的权利。区分已授权专利和已公开申请
- **空白空间误解**：空白区域可能意味着"没人尝试"或"人人尝试但失败"。在假设机会前先调查
- **图谱当法律意见**：此技能产出战略情报，不是法律建议。此处标记的 FTO 风险需要专利顾问的正式分析
